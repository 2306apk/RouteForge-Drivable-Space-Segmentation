# **Real-Time Drivable Space Segmentation**

## **Project overview**
This project tackles a fundamental challenge in Level 4 autonomous driving: identifying safe, drivable "Free Space" in complex urban environments without relying on painted lane markings. The objective is to perform highly accurate, pixel-wise semantic segmentation of drivable versus non-drivable areas (e.g., roads vs. sidewalks, curbs, and obstacles). 

To ensure safety-critical response times, the pipeline is heavily optimized for low-latency inference, achieving ~1000 FPS. Furthermore, to adhere to strict challenge constraints, the entire perception pipeline and neural network were developed and trained entirely from scratch without the use of pre-trained ImageNet weights or pre-built library architectures.

## **Model architecture**
The core of the perception engine is a custom-built **U-Net (Encoder-Decoder)** architecture designed for high-frequency binary segmentation.
* **Encoder (Contraction):** Utilizes three progressive stages of 3x3 Convolutional blocks, ReLU activations, and Max Pooling to extract deep semantic context and spatial hierarchies.
* **Bottleneck & Skip Connections:** Long-range skip connections concatenate high-resolution feature maps from the encoder directly into the decoder, preserving fine-grained boundary details (crucial for precise road-to-curb transitions).
* **Decoder (Expansion):** Employs learnable Transposed Convolutions to upsample the feature maps back to the original 128x128 spatial resolution.
* **Output Head:** A final 1x1 Convolution outputs raw logits, which are processed via a Sigmoid activation during inference to generate a binary mask (Threshold > 0.5).

## **Dataset used**
* **Primary Source:** **nuScenes (v1.0-mini)** dataset.
* **Input Data:** High-resolution RGB frontal camera frames (`CAM_FRONT`).
* **Ground Truth Generation:** Binary segmentation masks (Drivable vs. Non-Drivable) were generated by projecting the 3D spatial vector maps from the nuScenes relational database onto the 2D camera image plane.
* **Processing:** Images and masks were resized to 128x128 and normalized. The dataset was split 80/20 for training and validation to monitor Mean Intersection over Union (mIoU) on unseen data and prevent overfitting.

## **How to run the code**
**1. Training the Model:**
To train the custom U-Net from scratch (this will automatically process the dataset, apply the 80/20 validation split, and save the best `model.pth` weights):
```bash
python train.py
```

**2. Running Inference:**
To test the trained model on a sample image and visualize the FPS and segmentation overlay:
```bash
python inference.py
```

## **Example outputs / results**
The custom U-Net successfully distinguishes drivable asphalt from obstacles, parked cars, and sidewalks. 

* **Inference Speed:** ~1000 FPS on a standard T4 GPU backend.
* **Optimization:** Utilized Mixed Precision (`torch.amp`) and a hybrid BCE + Dice Loss function to achieve sharp boundary detection despite class imbalances.
